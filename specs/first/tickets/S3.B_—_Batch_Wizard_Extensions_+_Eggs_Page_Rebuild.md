### Component breakdown

| Component | File | Responsibility |
| --- | --- | --- |
| `Eggs` (page) | `pages/Eggs.tsx` | Orchestrator — batch picker, tab routing, query wiring |
| `BatchPicker` | `components/eggs/BatchPicker.tsx` | Layer + duck-layer batches only; shows species sub-badge |
| `ProductionHeader` | `components/eggs/ProductionHeader.tsx` | Rate vs expected, week-over-week trend, withdrawal banner |
| `CollectionDialog` | `components/eggs/CollectionDialog.tsx` | 6 grade inputs (S/M/L/XL/cracked/dirty); live good-egg preview |
| `SaleDialog` | `components/eggs/SaleDialog.tsx` | Crates + loose, price, buyer, payment method; total preview |
| `RecordsTable` | `components/eggs/RecordsTable.tsx` | Daily collection history with grade breakdown |
| `SalesTable` | `components/eggs/SalesTable.tsx` | Sale history with revenue (masked when privacy active) |

### Key frontend rules

- `BatchPicker` only shows `species='layer'` and `species='duck' AND duck_type='layer'` batches — enforced by filtering the batch list client-side (server enforces via 404)
- `ProductionHeader` shows a withdrawal banner when `egg-withdrawal-status.active = true`
- `CollectionDialog` has exactly 6 grade inputs — no "total eggs" shortcut field
- `SaleDialog` computes `total = crates * pricePerCrate + loose * pricePerLoose` client-side for preview only; server is authoritative
- Revenue values in `SalesTable` are displayed only when server returns them (privacy-aware — no client-side masking)
- All write actions use `Idempotency-Key` header; Phase 2+ writes are online-only (no offline queue)
