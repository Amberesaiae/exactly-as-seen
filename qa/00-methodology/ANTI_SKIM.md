# Anti-skim rules

## What skim looks like (banned as “audit complete”)

- One login, one batch, one feed click, dashboard screenshot  
- “RPC exists therefore sole path”  
- Trusting `PATH_TO_100` / PRODUCTION_E2E percentage  
- Playwright: dialog opens / body visible  
- Reading only happy-path toast  
- Declaring multi-species “done” after broiler only  
- Closing dual-pattern after intensive only  
- Treating offline as “Dexie tables exist”

## What depth requires

For **each** system pack:

| Layer | Minimum |
|-------|---------|
| Code inventory | Every write site for that domain (file:function) |
| Writer class | SPINE / KILL / FOLD / KEEP (per WRITER_MAP) |
| Usecases | Happy + fail-closed + dual-pattern + species where relevant |
| Live proof | UI + network intent + DB side-effects |
| Grey list | Explicit unknowns remaining |
| Curation | Fix, or accept residual with rationale |

## Time expectation

A **system pack** is not a 20-minute pass. Budget:

| Phase | Rough effort |
|-------|----------------|
| Inventory | hours, not minutes |
| Grey map | hours |
| Acid e2e (all greys) | half-day to multi-day per large system |
| Curation | as needed |

If it felt “quick,” it was skim. Re-open the pack.

## Sign-off

A pack may move to `CLOSED` only when the auditor records:

- inventory complete date  
- greys count = 0 or all deferred with ID  
- acid matrix green  
- second pair of eyes optional but recommended for money systems (C, D, E, F, G)
